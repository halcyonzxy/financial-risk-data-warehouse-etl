# financial-risk-data-warehouse-etl
企业级金融风控数仓ETL项目 | ODS-DWD-DWS 分层架构
项目简介
本项目是企业级金融风控数仓 ETL 全流程解决方案，基于 ODS → DWD → DWS 标准分层架构构建，专为期货 / 信贷类风控场景设计。项目从原始需求出发，彻底重构了数据模型、清洗逻辑、指标口径、监控体系，达到可直接上线的生产级标准。
项目亮点（企业最看重的点）
标准数仓分层：ODS 原始层 → DWD 明细清洗层 → DWS 聚合应用层
全表主键约束：所有表具备唯一主键，保证数据唯一性
ETL 幂等性：支持重复运行、重跑、补数据，不重复、不报错
真实业务口径：盈亏、持仓、交易量均使用真实业务逻辑
数据质量监控：自动捕获脏数据、缺失值、异常值、重复数据
风控监控体系：信用风险、市场风险、持仓风险实时输出
无硬编码、无虚假逻辑：所有指标可追溯、可审计
ODS 层：原始数据接入（客户信息、交易流水、市场风险指标）
DWD 层：数据清洗、去重、格式标准化、异常数据分流
DWS 层：业务指标聚合、风险监控、数据质量清单
核心功能模块
1. DWD 明细清洗层（核心清洗逻辑）
客户信息清洗（去重、空值修复、异常信用分处理）
交易数据合规校验（金额校验、交易量合法性）
异常交易自动分流
市场风险指标清洗
2. DWS 应用层（业务输出）
客户 - 品种日频业务指标表
持仓量、成交量、真实盈亏、信用等级、风险敞口
业务风险监控表
客户信用风险、市场指标超标风险、大额持仓风险
数据质量异常明细表
缺失客户 ID、金额不一致、交易量异常、重复交易
3. 全流程 ETL 保障
支持按日期增量跑数
支持重复执行、补数据
无主键冲突、无数据膨胀
所有口径可审计、可回溯
